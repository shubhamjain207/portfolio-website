Depth testing | OpenGL engine | 11/11/2024



Before and after applying depth testing, the fragments with depth values closer to the camera are drawn, and fragments behind them (with greater values) are discarded. When these values are multiplied with the final color in the fragment shader, we get a smooth shading effect.
In shaders, the depth value can be used creatively to achieve visual effects like smooth shading. For example, multiplying the fragment's final color with its depth value can create a gradient effect that simulates depth perception.

Torch Light | OpenGL engine | 22/06/2024

First we set up the light and its position, we want to project light on a fragment to give a torch effect, means light calculations in circular shape. Then we calculate a unit direction vector from object to light. Now we got many unit vectors which are pointing towards the light on each vertex. To simplify the explanation we are projecting the light downwards as light is above the object. So, the direction in which the light is aiming at the object will simply be (0.0,1.0,0.0). Now, we calculate the cosine of angle between these two unit vectors, by calculating their dot product.



There are two vectors a and b, pointing in lets say opposite direction, which makes the angle 180 degrees, then cosine of angle 180 would be -1, which means one vectors needs to be scaled by -1 to point in the same direction as the other one. We already have a cut off value which is a cos of angle, acting as a radius of light. If our calculated value is less than the cutoff value we compute the light we want in the torch else we can make it dark to give a torch effect.

Attenuation implementation | OpenGL engine | 15/06/2024

Attenuation means to decrease the light intensity on the fragment, when the distance between the fragment and the light increases. For applying attenuation, we use a formula - 1/constant value + linearValue * distance + quadraticValue * (distance)squared.



Constant value always remains same to avoid division by zero. If the linear value is more than quadratic light diminishes slowly by distance between fragment and light position because of the linear increment. It is most preferable in no artificial light settings like fog or smoke. If the quadratic value is more than linear, light diminishes quickly by distance because the increment is exponential. Used in point light system, for instance. This value is then multiplied with the final light value generally that of diffuse light.

Diffuse Lightning | OpenGL | 23/04/2024

In diffuse lightning, first we set up our normal vectors, which is just a unit vector perpendicular to the object. It is an addition to our Vertex's children namely position, color, texture. Then we calculate the direction the object is pointing towards light by subtracting light's position from the object's position. Moving on we calculate the dot product between both the vectors (Normals and direction vector) to get the angle, which is the diffusion strength. Finally, we multiply the diffusion strength with the light color to get the light reflection of each Vertex point. The portion with maximum angle (generally 90) reflects the most light, hence appear brighter

Voxel Terrain Generation - OpenGL | 29/03/2024

This is a "Voxel" terrain chunk generated with the help of Simplex noise. Here first we decide the length and breadth of the chunk, in which individual cubes will be rendered using instancing of course, otherwise it would hinder the fps. Then, a height for individual pillar is determined using "Simplex Noise" top_level_y = glm::simplex(glm::vec2{ i / 40.0f,j / 40.0f }) * 30, here two values need to be noted, 1)40.0f and 2)30. For example, the noise generated values 1 and 10 for pillar 1 and 2 respectively, it means there is a good amount of difference between them, in this case it will not give the shape of the mountain. To resolve this we divide i and j with some value to "smoothen" out the values so that the difference between the group of values become less and it gives a shape of a terrain. 30 here, is just to specify the height of pillars, as noise values are extremely small.

Instancing - OpenGL | 23/02/2024

Simulation of millions of objects (even less or more) at once without lagging. We set up an extra layout location in vertex shader for our data to pass through, (better than uniforms in my opinion). Then set up a VBO for the layout, set the sizes of vertices, give the data, which is usually an array of matrices or values, and send it to the buffer object. Although we can change the data by glBufferData(), (don't forget to change to GL_DYNAMIC_DRAW), changing in the sense we can perform transformations (rotation, translations, changing colors, etc).
